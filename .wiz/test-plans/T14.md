# T14 Test Plan — End-to-end happy-path integration

## Acceptance Criteria
A single vitest spec at `apps/backend/src/__tests__/e2e.test.ts` runs the full
contract using the real Fastify server with the fake face engine, asserting:

1. Organizer creates an event and publishes it.
2. Three photos uploaded — two contain face A, one contains face B.
3. Two participants join (one matches face A, one matches face B) and post selfies.
4. After draining the queue, `me/photos?eventId=` returns 2 photos for participant A and 1 photo for participant B.
5. Organizer's balance is exactly `100 - 3` (one debit per match: A in 2, B in 1).
6. All photos `status === 'ready'`.
7. `foyer-data` reports `participants=2`, `photos=3`, `distributions=3`.
8. Deleting participant A removes A from every photo's `matchedUserIds` and removes the selfie file.

## Test Scenarios
- Single integration test exercises steps 1-8 sequentially with assertions throughout.
