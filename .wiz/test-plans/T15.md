# T15 Test Plan — Demo seed script

## Acceptance Criteria
1. `apps/backend/scripts/seed-demo.ts` exports a `seedDemo({ db, storage, faceEngine })` helper that:
   - Drops any prior `Photi Demo` event (idempotency).
   - Creates the organizer user and the event `Photi Demo` (live status, brand colors).
   - Generates 20 placeholder JPEGs (sharp), tagged so the fake face engine can return per-photo vectors.
   - Creates three participant users + selfies, programmed to match a known subset of photos.
   - Drains the job queue.
   - Returns `{ organizerId, slug, participantIds[], photoIds[] }`.
2. CLI entry `seed:demo` script wired into `apps/backend/package.json` so `pnpm --filter backend run seed:demo` runs it against the real DB and prints the resulting ids/share link.
3. README "Demo seed" section calls out the command.
4. Vitest verifies the seed against an in-memory server.

## Test Scenarios (Vitest)
- **seedDemo against in-memory server**: returns 20 photoIds, 3 participantIds and a slug; after seeding, `/events/:slug/foyer-data` reports `participants=3`, `photos=20`, `distributions > 0`.
- **idempotency**: running seed twice ends up with one Photi Demo event (slug stable, no duplicates).
