# T18 Test Plan — Broadcast `photo-ready`

## Acceptance Criteria
1. `JobRunnerDeps` accepts `foyerHub: FoyerHub`. `buildServer` passes the shared hub.
2. `processPhoto` broadcasts `{ type: 'photo-ready', eventId, photoId, isFeatured }`:
   - On the no-match path after `status='ready'` is written.
   - On the matched-in-transaction path after the transaction commits.
   - **Not** on the `awaiting_credit` path.
3. Web Foyer SSE handler treats `photo-ready` and `photo-featured` the same — refetches `/foyer-data`.
4. No schema changes; broadcast is additive.

## Test Scenarios
- **backend processPhoto.test.ts** — add a spy `foyerHub.broadcast` and assert it fires `photo-ready` for a matched-photo run AND a no-match run, and **does not** fire on awaiting_credit.
- **web foyer** — synthesize a `photo-ready` SSE event and assert the foyer-data query is refetched (new featured arrives).
