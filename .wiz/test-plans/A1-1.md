# A1-1: Cover (kapak) image control on NewEvent + SettingsTab

## Acceptance criteria

- [x] `apps/web/src/pages/organizer/NewEvent.tsx` renders a control labelled "Kapak" so organizers can specify a cover image at create time.
- [x] On submit, the cover URL value is included in the `POST /events` request body when non-empty.
- [x] `apps/web/src/pages/organizer/tabs/SettingsTab.tsx` exposes the same cover image control so existing events can be edited.
- [x] `packages/shared/src/event.ts` `CreateEventInput` accepts an optional `coverImageUrl: string().url().optional()`.
- [x] Backend `POST /events` (`apps/backend/src/routes/events.ts`) persists `coverImageUrl` from the create payload onto the new event row.
- [x] Existing CreateEventInput consumers (omitting cover) keep working — additive change only.
- [x] A short comment in the form explains why the URL approach was chosen.

## Decision: URL field (option 2)

We picked the **URL** option, mirroring the existing `brandingLogoUrl` pattern. Rationale:
- Avoids creating a new multipart endpoint solely for a single image; keeps the diff small.
- Matches the existing logo URL field UX so organizers learn one pattern.
- The local storage adapter already exposes `/files/:key` for any uploaded asset; nothing prevents an organizer from pasting that URL.

## Test scenarios

1. **Web — NewEvent posts coverImageUrl** (`organizer-pages.test.tsx`)
   - Render `NewEvent`.
   - Mock `POST /events` and capture the request body.
   - Type a value into the "Kapak" URL input.
   - Submit. Assert that the captured POST body contains `coverImageUrl: "<typed value>"`.

2. **Web — SettingsTab patches coverImageUrl** (`organizer-pages.test.tsx`)
   - Render `SettingsTab` with an event whose `coverImageUrl` is initially empty.
   - Find the cover input by accessible label, type a URL.
   - Click "Kaydet". Assert `PATCH /events/e1` body contains `coverImageUrl`.

3. **Backend — POST /events persists coverImageUrl** (`events-route.test.ts`)
   - POST a `CreateEventInput` with `coverImageUrl: "https://cdn.example/c.jpg"`.
   - Assert the response body has `coverImageUrl` set.
   - Re-fetch via `GET /events/:slug` and assert the public projection still has it.

4. **Backwards compatibility** — existing tests that omit `coverImageUrl` must still pass (no schema-required field added).
