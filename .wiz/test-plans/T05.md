# T05 Test Plan — Event endpoints + slug + QR

## Acceptance Criteria
1. `POST /events` validates `CreateEventInput` (Zod), generates slug `kebab(title)-{6char}` with collision retry, persists, returns 201 + event row.
2. `GET /events/mine` returns events owned by current user (newest-first).
3. `GET /events/:slug` returns public-meta only (no `ownerId`, no internal fields).
4. `PATCH /events/:id` owner only (other users 403).
5. `POST /events/:id/publish` flips `status=live` (owner only).
6. `POST /events/:id/archive` flips `status=archived` (owner only).
7. `GET /events/:id/qr` returns PNG (`image/png`) encoding `${PUBLIC_BASE}/e/{slug}`.

## Test Scenarios (Vitest)
- **slug**: `slugify("Ali’nin Doğum Günü!")` produces a clean kebab-case ASCII slug; collision-retry returns a different slug when first roll exists.
- **POST /events**: creates with valid input, returns 201, id + slug present, slug matches `/^[a-z0-9-]+-[a-z0-9]{6}$/`.
- **POST /events**: returns 400 on schema failure (missing title).
- **GET /events/mine**: returns only the requesting user's events.
- **GET /events/:slug** (with `x-user-id` header): returns public meta; missing `ownerId` field.
- **PATCH /events/:id** as owner: updates title; returns 200.
- **PATCH /events/:id** as non-owner: returns 403.
- **POST publish/archive** as owner: status updated; non-owner 403; missing event 404.
- **QR**: returns 200, content-type `image/png`, body starts with PNG magic bytes.
