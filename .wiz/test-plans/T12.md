# T12 Test Plan — Participant Pages + Polish

## Acceptance Criteria
1. `EventLanding` shows cover/title and a "Katıl" CTA. Clicking it POSTs `/events/:slug/join` then navigates to `/e/:slug/selfie`.
2. `Selfie`: starts camera via `getUserMedia` (graceful fallback to `<input type=file capture>`), captures a frame onto a canvas, uploads multipart, shows ✓ screen on success, displays privacy note + "Verimi Sil" link that calls DELETE participant.
3. `Gallery`: polls `/me/photos?eventId=` every 5s, renders cards with download buttons; first time a match arrives, requests `Notification.permission` and on subsequent count deltas fires `new Notification(...)`.
4. `Foyer`: 6s rotation through featured photos (uses fake timers in test), SSE subscription, `F` toggles fullscreen, 30s watchdog reload.
5. A11y: focus rings, aria-labels, alt text, `prefers-reduced-motion` respected (no auto-rotate).
6. Backend Dockerfile builds + runs migrate + start.
7. README updated with the 3-command setup + the spec's end-to-end manual test scenario.

## Test Scenarios (Vitest)
- **EventLanding**: clicking Katıl calls join endpoint and navigates.
- **Selfie capture & upload**: camera mocked; clicking "Çek + Gönder" uploads via mocked multipart; success screen shows.
- **Selfie delete-my-data**: click DELETE link → calls DELETE participant.
- **Gallery notify**: with permission granted, advancing count from 1 → 2 fires `new Notification` once; with permission denied, no notification.
- **Foyer rotation**: with `vi.useFakeTimers()`, advancing 6s flips to next featured photo. Skips rotation when `prefers-reduced-motion: reduce`.
- **Foyer F key fullscreen**: pressing F calls `requestFullscreen` on the wrapper.
