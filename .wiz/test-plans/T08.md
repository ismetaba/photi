# T08 Test Plan — Participant join + selfie + my-photos + delete + billing

## Acceptance Criteria
1. `POST /events/:slug/join` upserts a `Participant` row keyed by (event, user) and returns the row. Idempotent.
2. `POST /participants/:id/selfie` (multipart, single file). Only the participant herself can post; otherwise 403. Runs face detect+embed inline:
   - 0 faces or >1 faces → 422 with `code=face_count`.
   - Stores selfie under `participants/{id}/selfie.jpg`.
   - Persists `faceVector` JSON.
   - Enqueues `match-selfie` job for participantId.
3. `GET /me/photos?eventId=X` returns photo list items where `matchedUserIds` contains the requesting userId. Public-shaped items (with signed URLs).
4. `GET /events/:id/participants` (owner) returns `ParticipantListItem[]` with selfie thumb URL (or null) and match count.
5. `DELETE /participants/:id` (own) removes selfie storage object, nulls `selfieKey`/`faceVector`, removes user from `matchedUserIds` of every photo in that event.
6. `GET /billing/packages` returns the three packages from shared `BILLING_PACKAGES`.
7. `POST /billing/purchase` `{ packageId }`: increments user balance, writes `purchase` PhotiTransaction (positive amount), enqueues `retry-awaiting` job for the user. 400 for unknown packageId.

## Test Scenarios (Vitest)
- **join**: 200, returns participant; second call returns same participant id (idempotent).
- **selfie 0/2 faces** → 422 `face_count`.
- **selfie 1 face**: stores selfie, faceVector populated, match-selfie job queued.
- **selfie non-owner** → 403.
- **/me/photos**: matched user gets photo; non-matched gets empty list.
- **/events/:id/participants**: owner only; non-owner 403; returns selfie thumb URL when set; matchCount counts photos where userId is in matchedUserIds.
- **DELETE participant**: storage cleared; faceVector null; user removed from photo.matchedUserIds.
- **billing/packages**: returns three known packages.
- **billing/purchase**: balance increases, purchase txn written, retry-awaiting job queued; unknown packageId 400.
