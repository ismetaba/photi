# T04 Test Plan — Local Storage Adapter + Files Proxy + Sharp Thumbnails + EXIF

## Acceptance Criteria
1. `StorageAdapter` interface with `putObject(key, buffer, contentType?)`, `getSignedUrl(key, ttl?)`, `getStream(key)`, `remove(key)`, `exists(key)`.
2. `LocalAdapter` writes binaries under a configurable `storageDir`, returning paths relative to that root. `getSignedUrl` returns `/files/${urlEncodedKey}`.
3. `GET /files/:key` streams the binary content with the correct `content-type`. Missing key → 404.
4. `services/thumbnail.ts` produces a 400px webp via sharp.
5. `services/exif.ts` returns ISO `takenAt` from a JPEG that has DateTimeOriginal; otherwise `null`.

## Test Scenarios (Vitest)
- **LocalAdapter**: `putObject` then `getStream` returns identical bytes. `exists` reflects state. `remove` deletes the file. `getSignedUrl` returns `/files/<encoded-key>`. Slashes in keys get url-encoded.
- **/files proxy**: Boot test server with a fixture key written; GET serves correct status, content-type and content-length matching bytes. Unknown key → 404.
- **Thumbnail**: feed a synthetic 800×600 PNG → output is webp magic bytes + width ≤ 400.
- **EXIF**: feed a JPEG with EXIF DateTimeOriginal → returns ISO string. JPEG without EXIF → `null`. Non-image buffer → `null`.
