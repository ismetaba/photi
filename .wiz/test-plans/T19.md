# T19 Test Plan — Mirror photo-ready broadcast

## Acceptance Criteria
1. `matchSelfie` broadcasts `photo-ready` for every photo whose status flips to `ready` inside its loop. No broadcast when the branch flips to `awaiting_credit`.
2. `retryAwaiting` either:
   - delegates to `process-photo` (current behavior — covered by T18 when the queue drains), OR
   - broadcasts directly. Existing implementation re-enqueues a `process-photo` job per stuck photo, so the broadcast comes from T18; we just need a doc note + a regression test that asserts `retryAwaiting` followed by a queue drain produces a `photo-ready` broadcast.
3. JobRunner passes `foyerHub` into matchSelfie deps the same way it does for processPhoto.

## Test Scenarios (Vitest)
- **matchSelfie broadcast on ready**: program a photo + participant pairing that succeeds, run matchSelfie with a mock `foyerHub.broadcast = vi.fn()`. Assert one broadcast with `type: 'photo-ready'`, the right `eventId`/`photoId`/`isFeatured`.
- **matchSelfie no broadcast on awaiting_credit**: setup with organizer balance 0; same mock; run; assert broadcast NOT called.
- **retryAwaiting → drain triggers photo-ready**: mark a photo `awaiting_credit`, top up organizer, call retryAwaiting (which re-enqueues process-photo), then runner.drain() with a mock foyerHub. Assert at least one `photo-ready` broadcast.
