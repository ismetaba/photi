# T10 Test Plan — Web foundation

## Acceptance Criteria
1. `lib/userId.ts` reads/writes `photi:userId` in localStorage; lazy-creates a UUID on first call (`crypto.randomUUID`).
2. `api/client.ts` fetch wrapper auto-injects `x-user-id`, exposes typed JSON `get/post/patch/delete`, plus a `postMultipart` helper. Base URL via `import.meta.env.VITE_API_BASE` (default `http://localhost:3000`).
3. Router has all 8 routes from the task description.
4. `OrganizerLayout`: TopBar with brand mark + `BalanceBadge` (shows `me.balance`) + "Yeni Etkinlik" button. `LowCreditBanner` renders when `balance < 10`.
5. `ParticipantLayout`: mobile-first, applies `--event-color` CSS variable from a passed `brandingColor` prop.
6. TanStack Query provider wired with sensible defaults.

## Test Scenarios (Vitest, jsdom)
- **userId**: first call creates a UUID and persists it; second call returns same value; localStorage records the value.
- **api client**: fetch is mocked; outgoing request has `x-user-id` header; JSON responses parsed; error responses throw with code.
- **api client multipart**: returns FormData wrapper, includes header.
- **LowCreditBanner**: renders text when balance < 10, hidden otherwise.
- **OrganizerLayout balance**: renders BalanceBadge with the provided balance.
- **ParticipantLayout**: applies brandingColor as CSS variable on container.
- **router smoke**: rendering `/` shows OrganizerLayout content (Home placeholder); rendering `/e/whatever` shows ParticipantLayout content.
