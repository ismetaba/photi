# T16 Test Plan — Upload mime-type validation

## Acceptance Criteria
1. Allow-list `image/jpeg | image/png | image/webp | image/heic | image/heif` in a single shared constant.
2. `POST /events/:id/photos`:
   - Each non-image file part is collected in `rejected: [{ filename, mimetype }]`.
   - If at least one valid image is created → 201 with `{ created, rejected }`.
   - If **all** parts are rejected → 415 `{ code: 'no_supported_images', rejected }`.
3. `POST /participants/:id/selfie`:
   - Non-image mimetype → 415 `{ code: 'unsupported_image_type', got }`. No DB write, no storage put.
4. `req.isMultipart()` 415 path untouched.
5. Web Dropzone surfaces filenames from `rejected[]` to the user (best-effort).

## Test Scenarios (Vitest)
- **photos**: upload 1 valid jpeg + 1 text/plain part → 201, body has `created.length===1`, `rejected[0].mimetype==='text/plain'`. Only one Photo DB row + storage objects exist.
- **photos**: upload only text/plain → 415 `no_supported_images`. No DB row.
- **selfie**: post application/octet-stream → 415 `unsupported_image_type`. Participant row's `selfieKey` and `faceVector` remain null. Storage adapter has nothing under `participants/:id/selfie.jpg`.
- **shared**: `IMAGE_MIME_ALLOWLIST` exported.
